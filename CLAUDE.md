# Policy Tracker — repo context for Claude

You are working inside SaferAI's policy tracker. Context you need:

## What this is

A weekly AI-policy monitoring system for SaferAI's policy team. Three digests (legislative, newsletter, papers) are generated by **Claude Code Routines** on Anthropic's cloud and committed to this repo. A static Astro site rebuilds on push to `main` and is served via GitHub Pages.

## What you may be here to do

1. **Running as a Routine** — you were invoked by one of the prompts in `routines/<type>/prompt.md`. Your job is to read `sources/*.yaml`, read the last 4 digests from `content/` to compute deltas, then write a new digest file to `content/YYYY-MM-DD/<type>-digest.md` and open a PR against `main`.

2. **Maintaining the tracker** — someone (Simon or a teammate) is editing sources, prompts, validators, or the site. Follow the conventions below.

## Conventions

- **Bun**, not npm/yarn. Scripts use `bun run ...`.
- **TypeScript**, strict.
- **Zod** for all schema validation.
- **Astro** for the site, no React.
- **No Anthropic API keys** anywhere in this repo — generation runs only inside Claude Code Routines using the subscription.
- **Every digest item has a primary source URL.** No uncited deadlines.
- **Every digest item has a `stage` field** (e.g., `consultation-open`, `trilogue`, `adopted`). Lifecycle position matters more than summary length.
- **Digests are deltas, not absolutes.** Read prior digests before writing. Say what moved.

## What not to do

- Do not add an Anthropic API key. Do not introduce billing outside the Team subscription.
- Do not rewrite `routines/legislative-digest/prompt.md` substantively without flagging it — it is the ported version of Chloé's existing Cowork task.
- Do not pad digests. Empty sections are better than filler.
- Do not commit to `main` directly when running as a Routine — always open a PR on a `claude/*` branch.
